# Функции в Python

## Доклад по дисциплине «Программирование на Python»

---

## Введение

Функции — одна из ключевых конструкций в Python и в программировании вообще. Они позволяют выделить повторяющийся код в отдельный блок, дать ему имя и вызывать по мере необходимости. Без функций любая более-менее серьёзная программа превратилась бы в нечитаемую простыню кода, которую невозможно поддерживать и отлаживать.

В этом докладе рассмотрим, как устроены функции в Python: от базового синтаксиса до продвинутых возможностей вроде декораторов и замыканий.

## 1. Определение и вызов функции

Функция в Python объявляется с помощью ключевого слова `def`, после которого идёт имя функции и круглые скобки с параметрами:

```python
def greet(name):
    print(f"Привет, {name}!")

greet("Марк")  # Привет, Марк!
```

Тело функции — это блок кода с отступом. Python не использует фигурные скобки, как C или Java, а полагается именно на отступы. Обычно это 4 пробела.

Если функция ничего явно не возвращает, она возвращает `None`.

## 2. Возвращаемое значение

Чтобы функция возвращала результат, используется оператор `return`:

```python
def square(x):
    return x ** 2

result = square(5)
print(result)  # 25
```

Можно возвращать несколько значений через запятую — Python автоматически упакует их в кортеж:

```python
def min_max(numbers):
    return min(numbers), max(numbers)

lo, hi = min_max([3, 1, 4, 1, 5])
print(lo, hi)  # 1 5
```

## 3. Параметры и аргументы

### 3.1. Позиционные и именованные аргументы

Аргументы можно передавать по позиции или по имени:

```python
def power(base, exp):
    return base ** exp

power(2, 10)        # позиционные
power(exp=10, base=2)  # именованные, порядок не важен
```

### 3.2. Значения по умолчанию

Параметрам можно задать значения по умолчанию. Тогда при вызове их можно не указывать:

```python
def greet(name, greeting="Привет"):
    print(f"{greeting}, {name}!")

greet("Алиса")            # Привет, Алиса!
greet("Алиса", "Здравствуй")  # Здравствуй, Алиса!
```

Важный момент: нельзя использовать изменяемые объекты (списки, словари) как значения по умолчанию. Это частая ошибка:

```python
# Плохо:
def add_item(item, lst=[]):
    lst.append(item)
    return lst

# lst будет один и тот же объект при каждом вызове!

# Правильно:
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst
```

### 3.3. *args и **kwargs

Иногда заранее неизвестно, сколько аргументов передадут. Для этого есть `*args` (произвольное число позиционных аргументов) и `**kwargs` (произвольное число именованных аргументов):

```python
def log(*args, **kwargs):
    print("Аргументы:", args)
    print("Именованные:", kwargs)

log(1, 2, 3, level="info", source="main")
# Аргументы: (1, 2, 3)
# Именованные: {'level': 'info', 'source': 'main'}
```

`args` — это кортеж, `kwargs` — словарь. Имена `args` и `kwargs` — просто соглашение, важны именно `*` и `**`.

## 4. Область видимости (scope)

Переменные внутри функции — локальные. Они не видны снаружи:

```python
def foo():
    x = 10
    print(x)

foo()       # 10
print(x)    # NameError: name 'x' is not defined
```

Функция может читать переменные из внешней области видимости, но не изменять их напрямую. Чтобы изменить глобальную переменную, нужно объявить её через `global`:

```python
counter = 0

def increment():
    global counter
    counter += 1
```

Но на практике `global` используется редко — это считается плохим стилем. Лучше передавать значения через аргументы и возвращать результат.

## 5. Лямбда-функции

Лямбда — это анонимная функция, записанная в одну строку:

```python
square = lambda x: x ** 2
print(square(4))  # 16
```

Она может содержать только одно выражение. Лямбды удобны как аргументы для `sorted`, `map`, `filter`:

```python
students = [("Алиса", 4.5), ("Боб", 3.8), ("Вера", 4.9)]
students.sort(key=lambda s: s[1], reverse=True)
# Отсортировали по среднему баллу, от большего к меньшему
```

Не стоит злоупотреблять лямбдами — если функция сложнее одного выражения, лучше определить обычную через `def`.

## 6. Рекурсия

Функция может вызывать саму себя. Это называется рекурсия:

```python
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

print(factorial(5))  # 120
```

Рекурсия хорошо подходит для задач, которые естественно разбиваются на подзадачи: обход деревьев, вычисление факториала, числа Фибоначчи. Но у Python есть ограничение на глубину рекурсии (по умолчанию 1000 вызовов), так что для больших данных лучше использовать итеративный подход.

## 7. Замыкания (closures)

Замыкание — это функция, которая запоминает переменные из окружающей области видимости, даже когда эта область уже завершилась:

```python
def make_multiplier(factor):
    def multiply(x):
        return x * factor
    return multiply

double = make_multiplier(2)
print(double(5))   # 10
print(double(10))  # 20
```

Функция `multiply` «замкнулась» на переменную `factor`. Замыкания часто используются для создания фабрик функций и в функциональном программировании.

## 8. Декораторы

Декоратор — это функция, которая принимает другую функцию и возвращает её модифицированную версию. Синтаксически это записывается через `@`:

```python
def log_call(func):
    def wrapper(*args, **kwargs):
        print(f"Вызов {func.__name__}")
        result = func(*args, **kwargs)
        print(f"Результат: {result}")
        return result
    return wrapper

@log_call
def add(a, b):
    return a + b

add(3, 4)
# Вызов add
# Результат: 7
```

Декораторы используются повсюду: логирование, кэширование, проверка прав доступа, замер времени. В стандартной библиотеке есть готовые декораторы, например `@functools.lru_cache` для кэширования результатов.

## 9. Аннотации типов

Начиная с Python 3.5, можно указывать типы параметров и возвращаемого значения. Это не влияет на выполнение, но помогает читать код и ловить ошибки через инструменты вроде mypy:

```python
def add(a: int, b: int) -> int:
    return a + b
```

Для более сложных типов используется модуль `typing`:

```python
from typing import List, Optional

def find_first(items: List[str], prefix: Optional[str] = None) -> Optional[str]:
    for item in items:
        if prefix is None or item.startswith(prefix):
            return item
    return None
```

## 10. Встроенные функции

Python предоставляет множество встроенных функций, которые не нужно импортировать: `print()`, `len()`, `range()`, `type()`, `int()`, `str()`, `list()`, `dict()`, `sorted()`, `map()`, `filter()`, `zip()`, `enumerate()` и другие.

Особенно полезны `map` и `filter` в связке с лямбдами:

```python
nums = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x**2, nums))     # [1, 4, 9, 16, 25]
evens = list(filter(lambda x: x % 2 == 0, nums))  # [2, 4]
```

Хотя в современном Python чаще предпочитают списковые включения (list comprehensions), так как они читабельнее:

```python
squared = [x**2 for x in nums]
evens = [x for x in nums if x % 2 == 0]
```

## Заключение

Функции в Python — мощный и гибкий инструмент. Даже базовый синтаксис `def` / `return` покрывает большинство задач, а продвинутые возможности вроде `*args`, замыканий и декораторов позволяют писать компактный и выразительный код. Главное — не усложнять: если задачу можно решить простой функцией с понятным именем и парой параметров, так и надо делать.
